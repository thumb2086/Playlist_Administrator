# 播放清單無法播放的終極解決方案

## 問題根源
Windows 11 的「媒體播放器」(Media Player) 和「Groove 音樂」是 UWP 應用程式，有嚴格的安全沙盒限制。當你點開 `Playlists` 資料夾中的 `.m3u8` 檔案時，播放器**無法自動獲得讀取隔壁 `Music` 資料夾的權限**，因此會出現 `0x80070005` (存取被拒) 錯誤。

## 解決方案（按推薦順序）

### 方案 A：使用 VLC Media Player（最推薦）✅
VLC 是傳統桌面應用程式，**沒有 UWP 的沙盒限制**，能完美處理相對路徑。

1. 下載並安裝 VLC：https://www.videolan.org/vlc/
2. 右鍵點擊任何 `.m3u8` 檔案 → 選擇「開啟檔案」→ 選擇「VLC media player」
3. 勾選「永遠使用此應用程式開啟 .m3u8 檔案」
4. 完成！之後所有歌單都能正常播放

### 方案 B：將 data 資料夾加入 Windows 媒體播放器的媒體櫃
1. 開啟「媒體播放器」應用程式
2. 點擊右上角的「設定」(齒輪圖示)
3. 選擇「音樂媒體櫃」→「新增資料夾」
4. 選擇你的 `data` 資料夾（整個資料夾）
5. 確認後，播放器就有權限讀取所有子目錄了

### 方案 C：手動解除檔案鎖定（已自動執行）
程式已經自動幫你執行了 PowerShell 的 `Unblock-File` 指令，清除了所有檔案的「來自其他電腦」標籤。

## 技術說明
- M3U8 檔案格式：✅ 正確
- 路徑格式：✅ 正確 (`../Music/歌曲.mp3`)
- 編碼：✅ UTF-8 with BOM
- 檔案存在性：✅ 所有音樂檔案都存在

**問題純粹是 Windows UWP 應用程式的安全限制，不是程式碼的問題。**

## 推薦做法
**強烈建議使用 VLC**，它是全球最流行的媒體播放器，對 M3U/M3U8 的支援最完善，而且完全免費開源。
